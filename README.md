| **R** Result | 수동 작업 대비 콘텐츠 제작 시간을 대폭 단축한 자동화 프로토타입 완성 및 디스코드를 통한 모바일 제어 환경 확보 |
| **K** Takeaway | 외부 의존성 관리 — n8n 업데이트 등 외부 환경 변화에 따른 시스템 가용성 유지의 중요성을 체감 / 후처리 공정의 필요성 — AI 생성 영상의 자연스러움을 극대화하기 위해 기술적 자동화 외에도 세밀한 후처리 고도화가 필요함을 확인 |

### Local LLM(Ollama) 연동형 OpenClaw 아키텍처 구축 및 최적화

**기술:** Docker, Ollama, OpenClaw, Llama 3, Claude 3.5

| 구분 | 내용 |
|------|------|
| **S** Situation | 오픈소스 AI 에이전트인 OpenClaw의 실무 적용 가능성을 검토하고, 상용 API 사용 시 발생하는 데이터 유출 리스크를 원천 차단하기 위한 폐쇄형 환경 연구 |
| **T** Task | Docker 기반 로컬 LLM(Ollama) 인프라를 구축하여 상용 모델 수준의 에이전트 기능 구현 여부를 검증하고, 운영 비용 및 보안성 측면의 장단점을 분석 |
| **A** Action | 컨테이너 기반 인프라 구축 — Docker로 Ollama와 OpenClaw를 통합 배포하고, 외부 통신 없이 독립적으로 작동하는 보안 에이전트 환경 구축 / 모델 성능 벤치마킹 — 로컬 모델(Llama 3 등)과 외부 상용 모델(Claude 3.5 등)의 추론 정확도 및 응답 속도를 다각도로 비교 테스트 / 리소스 효율성 분석 — 고사양 장비 상시 가동에 따른 전력 소모 및 하드웨어 유휴 리스크를 TCO 관점에서 분석 |
| **R** Result | 보안 이슈가 없는 완전 로컬 기반 에이전트 작동 확인. 다만, 장비 성능에 따른 속도 편차와 고사양 서버 유휴 상태 유지에 따른 비용 효율성 한계 도출 |
| **K** Takeaway | 엣지 컴퓨팅 접근 방식 — 단일 고사양 서버 의존 방식 대신, 모바일·저전력 기기 기반 경량 모델(SLM)을 활용한 분산 처리 아키텍처의 필요성 인식 / 비용 최적화 전략 — 네트워크 기반 기기 간 리소스 공유 및 엣지 컴퓨팅 접목 시, 보안성을 유지하면서 인프라 비용을 최소화하는 효율적인 AI 시스템 설계가 가능할 것으로 판단 |
